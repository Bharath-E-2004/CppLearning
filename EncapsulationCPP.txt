## 🔷 `this` Keyword in C++

### 👉 What is `this`?

`this` is a **pointer to the current object**.

It is used inside a class to refer to **the object that called the function**.

---

## 🔥 Why do we need `this`?

Main reason 👉 **name conflict** between data members and parameters.

---

### 🔹 Example WITHOUT `this` (problem)

```cpp id="1"
class Student {
    int age;
public:
    Student(int age) {
        age = age;   // ❌ wrong
    }
};
```

Here:

* Left `age` → parameter
* Right `age` → parameter
  Class variable is untouched.

---

### ✅ Correct using `this`

```cpp id="2"
class Student {
    int age;
public:
    Student(int age) {
        this->age = age;
    }
};
```

Now:

* `this->age` → object’s data member
* `age` → parameter

---

## 🔥 `this` with Member Function

```cpp id="3"
class Box {
    int length;
public:
    void setLength(int length) {
        this->length = length;
    }
};
```

---

## 🔥 Returning Current Object (`this`)

```cpp id="4"
class Test {
    int x;
public:
    Test& set(int x) {
        this->x = x;
        return *this;
    }
};

int main() {
    Test t;
    t.set(10).set(20);   // method chaining
}
```

## 🔷 Encapsulation

### 👉 What is Encapsulation?

**Encapsulation = Data hiding + Controlled access**

Means:

* Keep data **private**
* Access it using **public methods**

---

### 🔹 Why Encapsulation?

✔ Protect data
✔ Prevent misuse
✔ Improve maintainability
✔ Real-world modeling

---

## 🔥 Example WITHOUT Encapsulation (bad)

```cpp id="5"
class Student {
public:
    int marks;
};
```

Anyone can do:

```cpp id="6"
student.marks = -100;  // ❌ invalid
```

---

## ✅ Encapsulation (Correct Way)

```cpp id="7"
class Student {
private:
    int marks;

public:
    void setMarks(int m) {
        if(m >= 0 && m <= 100)
            marks = m;
    }

    int getMarks() {
        return marks;
    }
};
```

Now data is protected.

---

## 🔥 `this` + Encapsulation Together

```cpp id="8"
class Student {
private:
    int marks;

public:
    void setMarks(int marks) {
        this->marks = marks;
    }
};
```

`this` clearly separates **data member** from **parameter**.

---

## 🧠 Interview One-Liners

* `this` is a pointer to the current object
* Used to resolve name conflict
* Encapsulation hides data using private access
* Access to data is through public methods
* Improves security and control

## 🔥 Quick Question for You 😏

What will this print?

```cpp id="9"
class A {
    int x;
public:
    void set(int x) {
        x = x;
    }
    int get() {
        return x;
    }
};

int main() {
    A a;
    a.set(10);
    cout << a.get();
}
```

Think carefully 😉
(Answer: **garbage value**, because `this` not used)
